Prework Study

HTML

  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Both ID and Class elements are used with CSS later in the process
  • Think of HTMl as the framework for a webpage

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside of an element
  • inline CSS: quick permenant changes within HTML
  • internal style sheet: embed an entire stylesheet directly into HTML file
  • external CSS style sheet: keeps all CSS code in sepererate file to be linked in meta data
  • SELECTOR: defines the element or attribute to which the rules or declarations will apply
  • DECLARATION: contains property and value
  • the * selector adds properties to entire page/
  • you can combine two selectors with a comma

Git

  • [git status] prints details about current branch and staged changes
  • [git checkout -b BRANCH NAME] creates a new branch and switches to it
  • [git pull origin BRANCH NAME] pulls code from specified branch into local repo
  • [git add -A] Stages all changes to be pushed to github repo
  • [git commit -m "MESSAGE"] commits staged changes for repo push
  • [git push origin BRANCH NAME] pushes actual changes to github repo
  • [git touch FILE NAME] creates file
